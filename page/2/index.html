DAMA（十二）元数据管理

Posted on 2022-08-17 Edited on 2022-12-12

业务驱动因素

可靠且良好的元数据有助于：

1）通过提供上下文语境和执行数据质量检查提高数据可信度

2）通过扩展用途增加战略信息的价值

3）通过识别冗余数据和流程提高运营效率

4）纺织使用过时或不正确的数据

5）减少数据的研究时间

6）改善数据使用者和IT人员之间的沟通

7）创建准确的影响分析，从而降低项目失败风险

8）通过缩短系统开发生命周期时间缩短产品上线时间

9）通过全面记录数据背景、历史和来源，降低培训成本和员工流动影响

10）满足监管合规

元数据管理不善容易导致以下问题：

1）冗余的数据和数据管理流程

2）重复和冗余的字典、存储库和其他元数据存储

3）不一致的数据元素定义与数据滥用的相关风险

4）元数据的不同版本相互矛盾且有冲突，降低数据使用者信心

5）怀疑元数据和数据可靠性

目标与原则

目标包括：

  1. 记录和管理与数据相关的业务术语和知识体系，确保人们理解和使用数据的内容一致性
  2. 收集和整理来自不同来源的元数据，确保人们了解来自组织不同部门的数据之间的相似与差异
  3. 确保元数据的质量、一致性、及时性和安全
  4. 提供标准途径，使元数据使用者可以访问元数据
  5. 推广或强制使用技术元数据标准，以实现数据交换

实施元数据解决方案应遵循以下指导原则：

  1. 组织承诺。确保组织管理承诺，将元数据管理作为整体战略一部分，激昂数据作为企业资产进行管理
  2. 战略。制定元数据战略，考虑创建、维护、集成和访问元数据。元数据战略必须与业务优先级保持一致
  3. 企业视角。从企业视角确保未来可扩展性，通过迭代和增量交付实现，以带来价值
  4. 潜移默化。宣导元数据必要性和每种元数据用途；潜移默化将其价值鼓励业务使用元数据，同时为业务提供知识辅助
  5. 访问。确保员工了解如何访问和使用元数据
  6. 质量。元数据通常是通过现有流程生成，所以流程所有者对元数据的质量负责。
  7. 审计。制定、实施和审核元数据标准，简化元数据的集成和使用
  8. 改进。创建反馈机制，便于使用者将错误的、过时的元数据反馈给元数据管理团队。

基本概念

1.元数据与数据

元数据也是数据，用数据管理的方式进行管理。

元数据和非元数据划分边界：概念上看，与数据代表的抽象级别有关。 如：美国安全局对美国人使用电话的监控情况时，电话号码和童话时间通常被称为“元数据”，意味着“真实”数据只包括电话交谈的内容，常识时电话号码和通话时间也只是普通数据。

为了管理元数据，组织不应该担心理论上的区别，应该定义元数据需求，重点关注元数据能用来做什么（创建新数据、了解现有数据、实现数据间流转、访问数据、共享数据）和满足这些需求的源数据。

元数据标准和指南

度量指标

元数据管理环境的建议指标包括：

1）元数据存储库的完整性。企业元数据的理想覆盖率和实际覆盖率比较。

2）元数据管理成熟度。根据能力成熟度模型（CMM-DMM）成熟度评估方法，开发判断企业元数据成熟度指标

3）专职人员配备。通过专职人员的任命、整个企业的专职人员覆盖范围，以及职位描述中的角色定义说明，来评估组织对元数据的承诺

4）元数据使用情况。通过存储库的访问次数衡量用户对元数据存储库的使用情况和接受程度。

5）业务术语活动。使用、更新、定义解析、覆盖范围。

6）主数据服务数据遵从性。显示SOA解决方案中国年数据的中药情况。主数据服务上的元数据帮助开发人员决定新开发任务可以使用哪些现有服务。

7）元数据文档质量。一个质量指标是通过自动和手动两种方式评估元数据文档的质量。自动评估方式包括对两个源执行冲突逻辑的对比、测量二者匹配程度以及随时间推移的变化趋势。另一个度量指标是度量具有定义的属性的百分比，以及随时间推移而发生变化的趋势。手动评估方式包括基于企业质量定义进行随机或完整的调查。质量度量表明存储库中元数据的完整性、可靠性、通用性等。

8）元数据存储库克永霞。正常运行时间、处理时间。

DAMA（十一）数据仓库和商务智能

Posted on 2022-08-15 Edited on 2022-12-12

数据仓库，该技术赋能组织将不同来源的数据整合到公共的数据模型中去，整合后的数据能为业务运营提供洞察，为企业决策支持和创造组织价值开辟新的可能性。企业数据仓库提供了一种减少数据冗余，提高信息一直想，让企业能够利用数据做出更优决策的方法。

业务驱动因素

数仓建设的驱动力，主要是运营支持职能、合规需求和商务智能活动。越来越多的数据以越来越快的速度被使用，商业智能从回顾性评价发展到预测分析领域。

目标和原则

目标：

  1. 支持商业智能活动
  2. 赋能商业分析和高效决策
  3. 基于数据洞察寻找创新方法

原则：

  1. 聚焦业务目标。确保数仓用于组织最优先级的业务并解决业务问题
  2. 以始为终。让业务优先级和最终交付的数据范围驱动数仓内容的创建
  3. 全局性思考和设计，局部性的行动和建设。让最终的愿景指导体系架构，通过集中项目快速迭代构建增量交付，从而实现更直接的投资回报
  4. 总结并持续优化，不是一开始就这样做。以原始数据为基础，通过汇总和聚合来满足需求并确保性能，但不替换细节数据
  5. 提升透明度和自助服务。上下文信息越丰富，数据消费者越能从数据中获得更多数据价值。向利益相关方公开集成的数据及流程信息。
  6. 与数仓一起建立元数据。数仓成功的关键是能够准确解释数据。能回答基本问题，如“这个数字为什么是X”“这个是怎么计算出来的”“数据从哪里来的”。元数据的获取应该作为软件开发生命周期的一部分，元数据的管理也应该作为数仓持续运营的一部分。
  7. 协同。与其他数据活动协作，尤其是数据治理、数据质量和元数据管理活动
  8. 不要千篇一律。为每种数据消费者提供正确的工具和产品

基本概念

1.商业智能：第一层，指的是一种理解组织诉求和寻找机会的数据分析活动。数据分析的结果是用来提高组织决策的成功率。另一层，指的是支持这类数据分析活动的技术合集。决策支持工具，商业智能工具的不断进化，促成数据查询，数据挖掘，统计分析，报表分析，场景建模，数据可视化及仪表盘等一系列应用，被用于从预算到高级分析到方方面面。

2.数据仓库：两个组成部分，一个集成的决策支持数据库和与之相关的用于收集、清理、转换和存储来自各种操作和外部数据源数据的软件程序。还包括相依赖的数据集市，是数仓中数据子集的副本。广义上说，数仓包括为任何支持商业智能目标的实现提供数据的数据存储或提取操作。

3.数仓建设。指数仓中数据的抽取，清洗，转换，控制，加载等操作过程。数仓建设流程的重点，是通过强制业务规则，维护适当的业务数据关系，在运营的数据上视线一个集成的、历史的业务环境。还包括与元数据资料库交互的流程。

4.数仓建设方法。

“面向主题的，整合的，随时间变化的，相对稳定的支持管理决策的数据集合”，用规范化的关系模型来存储和管理数据

“为查询和分析定制的交易数据的副本”，多维模型

核心理念：

  1. 数仓存储的数据来自其他系统
  2. 存储行为包括以提升数据价值的方式整合数据
  3. 数仓便于数据被访问和分析使用
  4. 组织建设数仓，因为需要让授权的利益相关方访问到可靠的、集成的数据
  5. 数仓数据建设有很多目的，涵盖工作流支持、运营管理和预测分析

参考数据和主数据

引言

业务驱动因素

1）满足组织数据需求

2）管理数据质量

3）管理数据集成成本

4）降低风险

目标和原则

目标：

  1. 确保组织在各个流程中都拥有完整、一致、最新且权威的参考数据和主数据
  2. 促使企业在各业务单元和各应用系统见共享参考数据和主数据
  3. 通过采用标准的、通用的数据模型和整合模式，降低数据使用和数据整合的成本及复杂性

原则：

  1. 共享数据
  2. 所有权
  3. 质量
  4. 管理职责
  5. 控制变更。
  6. 权限

基本概念

1.主数据和参考数据的区别

​ 1）参考数据。例如：代码表和描述表，仅用于描述组织中其他的数据或者是仅用于将数据库中的数据和组织之外的信息联系起来

​ 2）企业结构数据。例如：会计科目表，能够按业务职责描述业务活动

​ 3）交易结构数据。例如：客户标识符，描述了交易过程中必须出现的一些要素

主数据：以与业务活动相关的通用和抽象概念形式提供业务活动预警的数据，包括业务交易中涉及的内部和外部对象的详细信息（定义和标识符），如客户、产品、雇员、供应商和受控域（代码值）

2.参考数据

3.主数据。

是关于业务实体（如雇员，客户，产品，金融结构，资产和位置等）的数据，这些实体为业务交易和分析提供语境信息。实体是客观世界的对象（人，组织，地方，或事物等）。实体被实体、实例以数据/记录的方式表示。

4.数据共享架构

活动

主数据管理活动

目标之一：根据组成主数据的属性来了解数据完整性

另一目标：了解数据质量

主数据管理的架构方法取决于业务战略、现有数据源平台以及数据本身，特别是数据血缘和波动性以及高延迟或低延迟影响。必须考虑数据消费和共享模型。维护工具取决于业务需求和架构选项。

主数据管理是数据整合过程。

工作必须有一个路线图，让各个系统可以把主数据值和标识符作为流程输入。在系统间建立单向的闭环，保持系统间值的一致性。

实施指南

遵循主数据架构

建立和遵循适当过的参考体系架构，对于管理和共享跨组织的主数据至关重要。

监测数据流动

涉及主数据和参考数据的数据整合过程，确保在组织内做到及时提取和分发数据。当数据在参考数据或主数据共享环境中流动时，应监控相关数据流，实现：

  1. 显示数据如何在整个组织中共享和使用
  2. 在管理系统和应用程序中识别数据的血缘关系
  3. 辅助进行问题根本原因的分析
  4. 展示数据整合和消费整合技术的有效性
  5. 通过数据消费展示源系统的数据值延迟
  6. 确定在集成组件中执行的业务规则和转换的有效性

管理参考数据变更

参考数据属于共享资源，不应该被任意变更。成功管理参考数据的关键在于，组织放弃本地控制共享数据的医院。维护这种支持，对参考数据的更改请求，提供一个接收和响应的通道。

数据修改的类型包括
1）对外部参考数据集的行级变更

2）外部参考数据集的结构变化

3）对内部参考数据集的行级变更

4）内部参考数据集的结构变化

5）创建新的参考数据集

变更请求流程：

接收变更请求 -> 确定利益相关方 -> 确认影响 -> 决策和沟通 -> 更新并通知（如果可行）

数据共享协议

为确保恰当的访问和使用，应建立共享协议，规定哪些数据可以共享，以及在何种条件下共享。

数据共享环境的负责人元有义务向下游数据消费者提供高质量数据，为履行这一职责，依赖于上游系统，。应建立服务水平协议（SLA）和指标，衡量共享数据的可用性和分享数据质量

参考数据和主数据治理

治理过程决定事项

  1. 要整合的数据源
  2. 要落实的数据质量规则
  3. 遵守使用规则的条件
  4. 要监控的行动和监控频率
  5. 优先级和数据工作响应等级
  6. 如何展示信息以满足利益相关方的需求
  7. 参考数据管理和主数据管理部署的标准授权扎口和预期。

度量指标

  1. 数据质量和遵从性
  2. 数据变更活动
  3. 数据获取和消费

  5. 数据管理专员覆盖率
  6. 拥有总成本
  7. 数据共享量和使用情况

文件和内容管理

引言

文件和内容管理是指针对存储在关系型数据库之外的数据和信息的采集、存储、访问和使用过程的管理。重点在于保持稳健和其他非结构化或半结构化信息的完整性，并使这些信息能够被访问

业务驱动因素

主要驱动因素包括：法规遵从性要求、诉讼响应能力和电子取证请求能力以及业务连续性要求。

提高效率，是改进文件管理的驱动力，文件管理方面的技术进步，有助于组织简化流程，管理工作流，消除重复性手工任务，并实现协作。

目标和原则

目标：

  1. 企鹅u包能够高速有效的采集和使用非结构化数据和信息
  2. 确保结构化和非结构化数据之间的整合能力
  3. 准守法律义务并达到客户预期

指导原则：

  1. 组织中的每个人，都应该在保护组织的未来方面发挥作用
  2. 档案和内容处理方面的专家应充分参与制度和规化的制定

档案保存指导原则:

  1. 问责原则
  2. 完整原则
  3. 保护原则
  4. 遵从原则

  6. 保留原则

数据集成和互操作

引言

数据集成是将数据整合成物理的或虚拟的一致格式。

数据互操作是多个系统间进行通信的能力。

数据集成和互操作的解决方案提供了大多数组织所以来的基本数据管理职能：

  1. 数据迁移和转换
  2. 数据整合到数据中心或数据集市
  3. 将供应商的软件包集成到组织的应用系统框架中
  4. 在不同应用程序或组织之间数据共享
  5. 跨数据存储库和数据中心分发数据
  6. 数据归档
  7. 数据接口管理
  8. 获取和接收外部数据
  9. 结构化和非结构化数据集成
  10. 提供运营智能话和管理决策支持

业务驱动因素

主要目的是为了对数据移动进行有效管理

目标和原则

目标：

  1. 及时以数据消费者所需的格式提供数据
  2. 将数据物理地或虚拟地合并到数据中心
  3. 通过开发共享模型和接口来降低管理解决方案的成本和复杂度
  4. 识别有意义的事件（机会和威胁），自动触发警报并采取相应行动
  5. 支持商务智能、数据分析、祝数据管理以及运营效率的提升

原则：

  1. 采用企业视角确保未来的可扩展性设计，通过迭代和增量交付实现
  2. 平衡本地数据需求与企业数据需求，包括支撑和维护
  3. 确保数据集成和互操作设计和活动的可靠性。

基本概念

1.抽取、转换、加载（ETL）

  1. 抽取。包括选择所需的数据并从其源数据中提取
  2. 转换。让选定的数据和目标数据库的结构相兼容
  3. 加载。在目标系统中物理存储或呈现转换结果
  4. 抽取、加载、转换（ELT）

  1. 映射。转换的同义词，从源结构到目标结构建立查找矩阵的过程，也是该过程的结果。

2.时延（Latency）

3.复制

4.归档

5.企业消息格式/规范格式

6.交互模型

7.数据集成和互操作架构概念

8.数据交换标准

数据安全和企业架构

安全架构涉及：

  1. 用户管理数据安全的工具
  2. 数据加密标准和机制
  3. 外部供应商和承包商的数据访问指南
  4. 通过互联网的数据传送协议

度量指标

  1. 安全实施指标

    1. 安全最新安全补丁企业计算机占比
    2. 安全并运行最新反恶意软件计算机占比
    3. 成功通过背调的新员工占比
    4. 年度安全实践测验中得分超过80%的员工占比
    5. 已完成正式风险评估分析的业务单位占比
    6. 发生如火灾、地震或其他灾难时，成功通过灾难恢复测试的业务流程占比

  2. 安全意识指标

    1. 风险评估结果
    2. 风险事件和配置文件
    3. 正式的反馈调查和访谈
    4. 事故复盘、经验教训和受害者访谈
    5. 补丁有效性审计

  3. 数据保护指标

    1. 特定数据类型和信息系统关键性排名
    2. 与数据丢失、危害或损坏相关的事故、攻击、盗窃或灾难的年损失预期
    3. 特定数据丢失的风险与某些类别受监管信息以及补救优先级排序相关
    4. 数据与特定业务流程的风险映射，与销售点设备相关的风险将包含在金融支付系统的风险预测中
    5. 对某些具有价值的数据资产及其传播媒介遭受攻击的可能性进行威胁评估
    6. 对可能意外或友谊泄漏敏感信息的业务流程中的特定部分进行漏洞评估

  4. 安全事件指标

    1. 检测到并阻滞了入侵尝试数量
    2. 通过防止入侵节省的安全成本投资回报

  5. 机密数据扩散

    应衡量机密数据的副本数量，减少扩散。机密数据存储位置越多，泄漏风险越大
