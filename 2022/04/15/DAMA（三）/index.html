数据治理

##3.1 引言

业务驱动因素

  1. 减少风险
    1. 一般性风险管理。洞察风险数据对财务或商誉的影响，包括对法律和监管的响应
    2. 数据安全。通过控制活动保护数据资产，可获得性，可用性，完整性，连续性，可审计和数据安全
    3. 隐私。通过制度和合规性监控，控制私人信息，机密信息，个人身份信息等
  2. 改进流程
    1. 法规遵从性
    2. 数据质量提升。通过真实可信等数据提升业务绩效的能力
    3. 元数据管理。建立业务术语表，定义和定位组织中的数据；确保组织中数据繁多的元数据被管理和应用
    4. 项目开发效率。在系统生命周期中改进，以解决整个组织的数据管理问题，包括利用数据全周期治理来管理特定数据的技术债务
    5. 供应商管理。控制数据处理的合同，包括云存储、外部数据采购、数据产品销售和数据运维

数据治理与IT治理区分开 。 IT治理制定关于IT投资、IT应用组合和IT吸纳光明组合决策，从另一个角度还包括软硬件和总体技术架构。IT治理的作用是确保IT战略、投资与企业目标、战略的一致性。COBIT框架

2.嵌入式

​ 不是一个附加管理流程，数据治理活动融合软件开发方法、数据分析应用、主数据管理和风险管理

原则导向

直到原则是数据治理，特别是数据治理策略的基础。把核心原则的阐述和最佳实践作为策略的一部分，参考这些原则可以减少潜在阻力。组织中会出现更多知道原则与相关的数据治理组件

##基本概念

以数据为中心的组织

以数据为中心的组织，将数据作为资产估值，在生命周期所有阶段进行管理，包括项目开发和持续运营阶段。业务处理的目标就是为了得到高质量数据。

数据治理组织

可参考政治治理角度来理解， 包括立法（定义策略，标准和企业架构），司法（问题管理和升级）和执法（保护和服务，管理职责）

通用的数据治理组织模型：

数据治理运营模型类型

集中管理模式中，数据治理组织监督所有业务领域中的活动

分布式管理模式中，每个业务单元中采用相同的数据治理运营模型和标准

联邦式管理模式中，数据治理组织与多个业务单元协同，以维护一致的定义和标准

数据管理职责

数据管理活动集中于以下：

  1. 创建和管理核心元数据
  2. 记录规则和标准
  3. 管理数据质量问题
  4. 执行数据治理运营活动

数据管理岗位的类型

  1. 首席数据管理专员。CDO替代角色，担任数据治理机构主席，也可以是虚拟组织或者在分布式数据治理组织中担任CDO，也可能是高层发起者

  2. 高级数据管理专员。数据治理委员会资深管理者

  3. 企业数据管理专员。负责监督跨业务领域数据职能

  4. 业务数据管理专员。业务领域专业人士，领域专家，对一个数据域负责。和利益相关方共同定义和控制数据

  5. 数据所有者。某个业务的数据管理专员，对其领域内数据有决策权

  6. 技术数据管理专员。某个知识领域内的IT专业人员，如：数据集成专家，数据库管理员，商务智能专家，数据质量分析师或元数据管理员

  7. 协调数据管理专员。领导并代表业务数据管理专员和技术数据管理专员进行跨团队或数据专员间的讨论

数据制度

包括数据治理管理的初衷的简要说明和基本规则，贯穿数据和信息的创造、获取、集成、安全、质量和使用的全过程。

数据制度是全局性的，支持数据标准以及数据管理和使用等关键方面的预期行为。

数据资产估值

是理解和计算数据对组织的经济价值的过程。

数据生命周期的大多数阶段涉及成本（获取、存储、管理和处置）。只有在使用时才有价值，同时还产生了与风险管理相关的成本。

当使用数据的经济效益超过了上述成本是时，就会显现价值。

其他度量价值的方式：

  1. 替换成本。灾难性数据破坏事件或数据中断时，数据替换或恢复成本，包括组织内的交易、域、目录、文档和指标信息等

  2. 市场价值。兼并或收购企业时作为企业资产的价值

  3. 发现商机。通过交易数据或通过售卖数据，从数据中发现商机获得的收入价值

  4. 售卖数据。一些组织为产品或销售将数据打包从数据中获得的洞察

  5. 风险成本。基于潜在罚款、补救成本和诉讼费用的估价。来自法律或监管的风险包括：a.缺少必要的数据；b.存在不应留存的数据; c.除上述成本外，包括数据不正确造成客户、公司财务和声誉受到损害; d.风险下降或风险成本的下降，其实是与提升和验证数据等操作干预成本等抵消之后的溢出部分。

##3.2 活动

规划组织的数据治理

执行就绪评估

  1. 数据管理的成熟度

  2. 变革能力

  3. 协作准备

  4. 与业务保持一致

####探索与业务保持一致

####制定组织触点

  1. 采购和合同
  2. 预算和资金
  3. 法规遵从性
  4. SDLC/开发框架

###制定数据治理战略

交付物：

  1. 章程
  2. 运营框架和职责
  3. 实施路线图
  4. 为成功运营制定计划

定义数据治理运营框架

需要考虑以下：

  1. 数据对组织的价值
  2. 业务模式
  3. 文化因素
  4. 监管影响

制定目标、原则和制度

制度可以包含多个不同方面内容：

  1. 由数据治理办公室认证确认组织用到的数据
  2. 由数据治理办公室批准成为业务拥有者
  3. 业务拥有者将在其业务领域委派数据管理专员，日常职责是协调数据治理活动
  4. 尽可能提供标准化报告、仪表盘或计分卡，满足大部分业务需求
  5. 认证用户将授予访问相关数据的权限，以便查询即席报表和使用非标准报告
  6. 定期复评所有认证数据，评价其准确性、完整性、一致性、可访问性、唯一性、合规性和效率等

推动数据管理项目

参与变革管理

组织需要一个团队负责：

  1. 规划。规划变革管理
  2. 培训。建立和执行数据治理项目培训
  3. 影响系统开发。与PMO合作，在软件开发生命周期中增加数据治理步骤
  4. 制度实施。
  5. 沟通。提高数据管理专员和其他数据治理专员对自身角色和职责 以及数据管理项目目标和预期认知。

沟通至关重要。沟通如下重点：

  1. 提升数据资产价值
  2. 监控数据治理活动搞得反馈并采取行动
  3. 实施数据管理培训
  4. 从关键领域衡量变革管理程度。a.意识到需要改变；b.希望参与并支持变革；c.知道如何改变；d.具备实施新技能和行为的能力；e.保持持续变革
  5. 实施新的指标和关键绩效

参与问题管理

问题管理是识别、量化、划分优先级和解决与数据治理相关问题的过程：

  1. 授权
  2. 变更管理升级
  3. 合规性
  4. 冲突
  5. 一致性
  6. 合同
  7. 数据安全和身份识别
  8. 数据质量

####评估法规遵从性要求

实施数据治理

高优先级前期工作：

  1. 定义可满足高优先级目标的数据治理流程
  2. 建立业务术语表，记录术语和标准
  3. 协调企业架构师和数据架构师，帮助他们更好的理解数据和系统
  4. 为数据资产分配财务价值，以实现更好的决策，并提高对数据在组织成功中所起的作用理解

发起数据标准和规程

数据管理知识领域内标准化概念示例：

  1. 数据架构。企业级数据模型，工具标准，系统命名规范
  2. 数据建模和设计。数据模型管理程序，数据模型命名规范，定义标准，标准域，标准缩写等
  3. 数据存储和操作。标准工具，数据库恢复和业务连续性标准，数据库性能，数据留存和外部数据采集
  4. 数据安全。数据访问安全标准，监控和审计程序，存储安全标准和培训需求。
  5. 数据集成。用于数据集成和数据互操作的标准方法、工具
  6. 文件和内容。包含内容管理标准及程序，包括企业分类法的使用，支持法律查询，文档和电邮保留期限，电子签名和报告分发方法
  7. 参考数据和主数据。数据管理控制流程，数据记录系统，建立标准及授权应用，实体解析标准
  8. 数据库仓库和商业智能。包括工具标准，处理标准和流程，报告和可视化格式标准，大数据处理标准
  9. 元数据。业务和技术元数据，元数据的集成和使用流程
  10. 数据质量。数据质量规则、标准测量方法、数据补救标准和流程
  11. 大数据和数据科学。数据源识别、授权、获取、记录系统、共享和刷新

制定业务术语表

目标：

  1. 对核心业务概念和术语有共同理解
  2. 降低由于对业务概念理解不一致导致数据误使用的风险
  3. 改进技术资产与业务组织之间的一致性
  4. 最大限度提高搜索能力，并能获得记录在案的组织知识。

度量指标

价值

  1. 对业务目标的贡献
  2. 风险降低
  3. 运营效率提升

有效性

  1. 目标的实现
  2. 扩展数据管理专员使用的相关工具
  3. 沟通有效性
  4. 培训有效性
  5. 采纳变革的速度

可持续性

  1. 制度和流程的执行情况（是否正常工作）
  2. 标准和规程的遵从情况（员工是否在必要时遵守指导和改变行为）

基于业务需求， 制定好治理的目标，实施过程中的指导原则，明确各组织、人员的分工，使用一定的工具，渐进式实施。并且有明确的度量指标。
