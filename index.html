DAMA（十）参考数据和主数据

Posted on 2022-07-18 Edited on 2022-08-15

参考数据和主数据

引言

业务驱动因素

1）满足组织数据需求

2）管理数据质量

3）管理数据集成成本

4）降低风险

目标和原则

目标：

  1. 确保组织在各个流程中都拥有完整、一致、最新且权威的参考数据和主数据
  2. 促使企业在各业务单元和各应用系统见共享参考数据和主数据
  3. 通过采用标准的、通用的数据模型和整合模式，降低数据使用和数据整合的成本及复杂性

原则：

  1. 共享数据
  2. 所有权
  3. 质量
  4. 管理职责
  5. 控制变更。
  6. 权限

基本概念

1.主数据和参考数据的区别

​ 1）参考数据。例如：代码表和描述表，仅用于描述组织中其他的数据或者是仅用于将数据库中的数据和组织之外的信息联系起来

​ 2）企业结构数据。例如：会计科目表，能够按业务职责描述业务活动

​ 3）交易结构数据。例如：客户标识符，描述了交易过程中必须出现的一些要素

主数据：以与业务活动相关的通用和抽象概念形式提供业务活动预警的数据，包括业务交易中涉及的内部和外部对象的详细信息（定义和标识符），如客户、产品、雇员、供应商和受控域（代码值）

2.参考数据

3.主数据。

是关于业务实体（如雇员，客户，产品，金融结构，资产和位置等）的数据，这些实体为业务交易和分析提供语境信息。实体是客观世界的对象（人，组织，地方，或事物等）。实体被实体、实例以数据/记录的方式表示。

4.数据共享架构

活动

主数据管理活动

1.识别驱动因素和需求

驱动因素通常包括改善客户服务和/或运营效率，以及减少与隐私和法律法规有关的风险。

2.评估和评价数据源

目标之一：根据组成主数据的属性来了解数据完整性

另一目标：了解数据质量

3.定义架构方法

主数据管理的架构方法取决于业务战略、现有数据源平台以及数据本身，特别是数据血缘和波动性以及高延迟或低延迟影响。必须考虑数据消费和共享模型。维护工具取决于业务需求和架构选项。

4.建模主数据

主数据管理是数据整合过程。

5.定义管理职责和维护过程

6.建立治理制度，推动主数据使用

工作必须有一个路线图，让各个系统可以把主数据值和标识符作为流程输入。在系统间建立单向的闭环，保持系统间值的一致性。

参考数据管理活动

1.定义驱动因素和需求

主要驱动因素是运行效率和更高的数据质量

2.评估数据源

3.定义架构方法

在购买和构建管理参考数据工具之前，关键是考虑管理参考数据的要求和塔带来的挑战。如：数据波动性，更新频率，消费模型。

4.建模参考数据

5.定义管理职责和维护流程

某些情况下，数据管理员直接负责参考数据的实际维护；在其他情况下，他们可能会负责推动此进程。

6.建立参考数据治理制度

工具和方法

主数据管理需要一些专门被设计用于实现标识管理的工具。 主数据管理可以通过数据整合工具，数据修复工具，操作型数据存储（ODS），数据共享中心（DSH）或专门的主数据管理应用来实现。

实施指南

遵循主数据架构

建立和遵循适当过的参考体系架构，对于管理和共享跨组织的主数据至关重要。

监测数据流动

涉及主数据和参考数据的数据整合过程，确保在组织内做到及时提取和分发数据。当数据在参考数据或主数据共享环境中流动时，应监控相关数据流，实现：

  1. 显示数据如何在整个组织中共享和使用
  2. 在管理系统和应用程序中识别数据的血缘关系
  3. 辅助进行问题根本原因的分析
  4. 展示数据整合和消费整合技术的有效性
  5. 通过数据消费展示源系统的数据值延迟
  6. 确定在集成组件中执行的业务规则和转换的有效性

管理参考数据变更

参考数据属于共享资源，不应该被任意变更。成功管理参考数据的关键在于，组织放弃本地控制共享数据的医院。维护这种支持，对参考数据的更改请求，提供一个接收和响应的通道。

数据修改的类型包括
1）对外部参考数据集的行级变更

2）外部参考数据集的结构变化

3）对内部参考数据集的行级变更

4）内部参考数据集的结构变化

5）创建新的参考数据集

变更请求流程：

接收变更请求 -> 确定利益相关方 -> 确认影响 -> 决策和沟通 -> 更新并通知（如果可行）

数据共享协议

为确保恰当的访问和使用，应建立共享协议，规定哪些数据可以共享，以及在何种条件下共享。

数据共享环境的负责人元有义务向下游数据消费者提供高质量数据，为履行这一职责，依赖于上游系统，。应建立服务水平协议（SLA）和指标，衡量共享数据的可用性和分享数据质量

组织和文化变革

参考数据和主数据治理

治理过程决定事项

  1. 要整合的数据源
  2. 要落实的数据质量规则
  3. 遵守使用规则的条件
  4. 要监控的行动和监控频率
  5. 优先级和数据工作响应等级
  6. 如何展示信息以满足利益相关方的需求
  7. 参考数据管理和主数据管理部署的标准授权扎口和预期。

度量指标

  1. 数据质量和遵从性
  2. 数据变更活动
  3. 数据获取和消费
  4. 服务水平协议（SLA）
  5. 数据管理专员覆盖率
  6. 拥有总成本
  7. 数据共享量和使用情况

文件和内容管理

引言

文件和内容管理是指针对存储在关系型数据库之外的数据和信息的采集、存储、访问和使用过程的管理。重点在于保持稳健和其他非结构化或半结构化信息的完整性，并使这些信息能够被访问

业务驱动因素

主要驱动因素包括：法规遵从性要求、诉讼响应能力和电子取证请求能力以及业务连续性要求。

提高效率，是改进文件管理的驱动力，文件管理方面的技术进步，有助于组织简化流程，管理工作流，消除重复性手工任务，并实现协作。

目标和原则

目标：

  1. 企鹅u包能够高速有效的采集和使用非结构化数据和信息
  2. 确保结构化和非结构化数据之间的整合能力
  3. 准守法律义务并达到客户预期

指导原则：

  1. 组织中的每个人，都应该在保护组织的未来方面发挥作用
  2. 档案和内容处理方面的专家应充分参与制度和规化的制定

档案保存指导原则:

  1. 问责原则
  2. 完整原则

​ 4）既有模式

​ 5）内容主题

（3）内容建模 将逻辑内容概念转化为家也有关系的内容类型，属性和数据类型的过程。内容建模有两个层次，一、信息产品级别，产粗一个像网站一样的实际可交付成果；二、组件级别，进一步详细说明构成信息产品模型的元素

DAMA（八）数据集成与互操作

Posted on 2022-06-15 Edited on 2022-07-07

数据集成和互操作

引言

数据集成是将数据整合成物理的或虚拟的一致格式。

数据互操作是多个系统间进行通信的能力。

数据集成和互操作的解决方案提供了大多数组织所以来的基本数据管理职能：

  1. 数据迁移和转换
  2. 数据整合到数据中心或数据集市
  3. 将供应商的软件包集成到组织的应用系统框架中
  4. 在不同应用程序或组织之间数据共享
  5. 跨数据存储库和数据中心分发数据
  6. 数据归档
  7. 数据接口管理
  8. 获取和接收外部数据
  9. 结构化和非结构化数据集成
  10. 提供运营智能话和管理决策支持

业务驱动因素

主要目的是为了对数据移动进行有效管理

目标和原则

目标：

  1. 及时以数据消费者所需的格式提供数据
  2. 将数据物理地或虚拟地合并到数据中心
  3. 通过开发共享模型和接口来降低管理解决方案的成本和复杂度
  4. 识别有意义的事件（机会和威胁），自动触发警报并采取相应行动
  5. 支持商务智能、数据分析、祝数据管理以及运营效率的提升

原则：

  1. 采用企业视角确保未来的可扩展性设计，通过迭代和增量交付实现
  2. 平衡本地数据需求与企业数据需求，包括支撑和维护
  3. 确保数据集成和互操作设计和活动的可靠性。

基本概念

1.抽取、转换、加载（ETL）

  1. 抽取。包括选择所需的数据并从其源数据中提取
  2. 转换。让选定的数据和目标数据库的结构相兼容
  3. 加载。在目标系统中物理存储或呈现转换结果
  4. 抽取、加载、转换（ELT）

  1. 映射。转换的同义词，从源结构到目标结构建立查找矩阵的过程，也是该过程的结果。

2.时延（Latency）

3.复制

4.归档

5.企业消息格式/规范格式

6.交互模型

7.数据集成和互操作架构概念

8.数据交换标准

    4. 数据与特定业务流程的风险映射，与销售点设备相关的风险将包含在金融支付系统的风险预测中
    5. 对某些具有价值的数据资产及其传播媒介遭受攻击的可能性进行威胁评估
    6. 对可能意外或友谊泄漏敏感信息的业务流程中的特定部分进行漏洞评估

  4. 安全事件指标

    1. 检测到并阻滞了入侵尝试数量
    2. 通过防止入侵节省的安全成本投资回报

  5. 机密数据扩散

    应衡量机密数据的副本数量，减少扩散。机密数据存储位置越多，泄漏风险越大

DAMA（六）数据存储和操作

Posted on 2022-06-13

数据存储和操作

引言

数据存储和操作，包括对存储数据的设计、实施和支持，最大化实现数据资源的价值，贯穿于数据创建/获取到处置的整个生命周期

1.数据库操作支持

关注于数据生命周期相关的活动，从数据库环境的初始搭建，到数据的获取，备份再到处置数据， 同时还包括需要确保数据库性能状态良好。监控和优化数据库性能，对数据库支持非常重要

2.数据库技术支持

包括定义满足组织需要的数据库技术要求，定义数据库技术架构，安装和管理数据库技术，解决和数据库相关的技术问题。

目标和原则

目标：

  1. 在整个数据生命周期中管理数据的可用性
  2. 确保数据资产的完整性
  3. 管理数据交易事务的性能。

原则：

  1. 识别自动化的机会并采取行动
  2. 构建时考虑重用思想
  3. 理解并适当使用最佳实践
  4. 支持数据库的标准需求
  5. 为项目中的dba角色设置期望值
